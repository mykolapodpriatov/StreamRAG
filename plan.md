# План реализации проекта StreamRAG

## 1. Идея и описание
**StreamRAG** – RAG-система для потоковых новостей и финансовых данных.
Система собирает данные из непрерывных потоковых источников (RSS-ленты, новостные сайты, официальные сообщения компаний, соцсети) и немедленно интегрирует их в векторную базу знаний. Позволяет получать ответы на основе актуальной информации.

## 2. Уникальные особенности
- **Ингесторы потоковых данных:** веб-краулеры и API для интеграции RSS, Reddit, открытых финансовых лент в реальном времени.
- **Механизм весов "time-decayed":** новые события получают больший приоритет при формировании контекста, при этом старые данные сохраняются для ретроспективного анализа.
- **Витрина аналитики:** встроенный дашборд для мониторинга времени добавления данных, объемов и источников.
- **Подписка на темы:** фильтрация контекста по темам (акции, макроэкономика) и использование специализированных финансовых эмбеддингов.
- **Политика хранения данных:** настройка TTL (time-to-live) для старых векторов в БД для контроля за объемом хранилища.

## 3. Выбор технологий
- **Фреймворк RAG:** LangChain.
- **Сбор данных:** feedparser (для RSS), BeautifulSoup, API открытых источников (Reddit), Celery (с Redis) для асинхронных задач.
- **Векторная БД:** Qdrant (из-за поддержки кастомного payload scoring и частых обновлений).
- **Модели:** LLM: gpt-4o-mini (или аналогичная), Эмбеддинги: text-embedding-3-small (или специализированная финансовая модель).
- **Оценка:** Ragas, Open RAG Eval.

## 4. Поэтапный план разработки

### Этап 1. Разработка асинхронного сбора данных
- Развернуть инфраструктуру очереди задач (message broker, например, Redis) для асинхронной обработки.
- Написать воркеры (ингесторы) для сбора данных из RSS и API с реализацией механизмов дедупликации записей.
- Реализовать очистку текста, первичную обработку потока, а также определение структур данных и тегирование метаданных для подписки на темы.

### Этап 2. Векторизация и управление контекстом
- Настроить механизм динамического обновления векторной БД.
- Настроить фильтрацию контекста по темам и подключить специализированные финансовые эмбеддинги.
- Реализовать логику `time-decayed` с использованием Qdrant custom payload scoring для убывания веса документа со временем при поиске.
- Разработать логику приложения RAG: пайплайн запросов (query pipeline), интеграция с LLM, а также UI/API для взаимодействия с пользователями.

### Этап 3. Аналитика и мониторинг
- Создать дашборд (например, на Streamlit), показывающий статистику по индексу и потокам данных.

### Этап 4. Тестирование и оценка
- Написать unit-тесты для ингесторов, логики очистки данных, синхронизации с векторной БД и алгоритма time-decay.
- Сравнить качество ответов системы на актуальные вопросы с использованием статических и потоковых баз данных.
- Оценить метрики качества с помощью Ragas.

### Этап 5. Документация и публикация
- Написать инструкции по развертыванию потоковых воркеров.
- Опубликовать проект на GitHub.
